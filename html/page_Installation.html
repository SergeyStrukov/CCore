CCore installation

Cook book

In order to use CCore, you have to "cook" it. The following ingredients are required:

  • Cygwin,
  • compiler set,
  • Eclipse (optional),
  • CCore package (or repository).

Install cygwin first. Include make and bash in your installation.
If you are not familiar with this platform spent some time to learn it.

Unpack the compiler set (provided as the archive opt.7z) in the cygwin root.
You will need 7-Zip archive manager. You will get the opt directory with several gcc compilers.
You can also build compilers from the sources yourself. See below for instructions.

Copy 4 dlls from the directory /opt/gcc-4.7.0/bin (or from another newer host compiler directory) to the /bin directory (all pathes here are cygwin pathes, dlls are cyggcc_s-1.dll, cygssp-0.dll, cygstdc++-6.dll, cygquadmath-0.dll).
It is neccessary to make host programs runable.

Create the directory bin in your cygwin home directory. CCore uses some tools and utilities, they are built in and invoked from this directory. Make sure this directory is included in the cygwin path.

Unpack CCore package (or checkout the source tree from a repository) somewere under you cygwin home directory. Avoid file and directory names with space characters.

Open bash console (or another console you would prefer, just configure it to use cygwin bash as a command processor).

Go to the CCore root directory. The following commands are used to prepare, clean and build the whole installation:

  • make — to build all,
  • make warn — to build all and put errors/warnings into warn.txt file,
  • make clean — to clean,
  • make list — to build source file lists; you will need this command if you alter source file set, before using it you have, however, build CCore tools.

Post-install notes

Once you install and build CCore, tools are built in the ~/bin directory. Two of them: MakeList.exe and Regen.exe, are the essential part of the build process, so keep them in the working condition.

You can use Eclipse as an IDE to work with CCore and CCore-based projects. CCore package includes the archive eclipse.7z with the Eclipse Indigo workbench. You will need Anonymouse Pro font for text editor. Before using this workbench you have to adjust some settings.
Open menu Window/Preferences... . Then open the C++/Build/Environment tab. Correct the variable CCORE_ROOT to point to the CCore root directory.
Then open the General/Workspace/Linked Resources tab and correct the path variable Root again to the CCore root directory.
There is a runable project in the workbench: WIN32-test. You have to adjust pathes in the project launch configuration also. Right-click on the project, pick the Properties dialog and and select Run/Debug Settings tab. Open the launch configuration for edit. Correct pathes in the Main and Arguments tabs.
You likely would have to refresh linked resources, select each project and press F5. Finally, customize Eclipse as you like.
