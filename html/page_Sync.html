Synchronization objects

Multi-threaded applications are impossible without a thread execution coordination. This is implemented using synchronization objects. Synchronization class has a set of methods, some of them may block execution of the calling thread, until some condition is met.

Most of synchronization object in CCore can be named in constructor, using a TextLabel. If the name is not given explicitly, then some auto-generated name is used. In HCore the provided name is ignored, but in XCore it is used by the system log and event recorder. You can get the name of the synchonization object in XCore by the method getName().

Blocking methods usually have four variants: try-variant which don't block, variant without time-out, with MSec time-out and with TimeScope time-out. The variant without timeout always successull (but may block indefinitely). Other variants returns the bool return value, which indicates if the method was successfull.

Releasing methods in XCore usually have three variants: common variant can be called only in a task context, "_int" variant can be called only in the interrupt context and "_any" variant can be called in any context. Also these methods have associated methods, which returns a Function object, bound to the call of the correspondent method.
