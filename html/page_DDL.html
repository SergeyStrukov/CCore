DDL is a Data Definition Language. This language is to record a general-kind data in a textual form similar to the C language. But there are important differences between how DDL interprets data and expresions and C. DDL is a "bye-bye XML" language.

Informal introduction

Documents txt/DDL-1*.odt contains number of precise definitions, related to DDL in a waterless form. An here is an informal introduction to the language.

DDL is to define named values. Each value has a type. Types can be basic and derived. The most important derived types are structures.

Basic types are: integral, text and ip. Intergral types are: 


uint8    sint8
uint16   sint16
uint32   sint32
uint64   sint64

(sint,uint) = (sint32,uint32) or (sint64,uint64)

int = sint

ulen = uint32 or uint64 >= uint

Text type represents an 8-bit character strings. ASCII encoding must be used at least for the first 128 code positions. Ip type represents an IPv4 addresses.

Derived types are: pointers, arrays and structures: 


int a = 10 ;

int * pa = &a ;

text [a] B = { "b1" , "b2" } ;

text [] C = { "c1" , "c2" } ;

struct S
 {
  text name = "unnamed" ;
  int id = 0 ;
 };

S record = { "" , 10 } ;
