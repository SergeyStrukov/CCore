Swap idiom

Files CCore/inc/Swap.h CCore/src/Swap.cpp

Swap is not just a set of functions and classes, but a fundamental idiom. Swapping means exchanging values as the one operation. Objects of a Small Data Types can be copied efficiently. In fact, it is a generalized definition of the such kind of types. Efficiency can be defined in another terms as a O(1)-nothow property. But for many types (actually, for the most of them) copying is either not logically possible, or is not efficient, because requires duplication of a large state, and that is time and space expensive and may fail. So instead of copying we should use moving or swaping whenever possible. For example, to return a result from a function: 

void func(T &ret)
 {
  T result;

  ....

  // result is built, took some resources

  Swap(result,ret); // result -> ret, ret -> result

  // null state result is destoyed
 }

void other_func()
 {
  ....

  T obj; // null state, no resource taken

  func(obj);

  ....
 }

Swap is derivable: if the state is a tuple of states and each of them is swapable, then the state is swapable. 

struct S
 {
  A a;
  B b;
  C c;

  // swap objects

  void objSwap(S &obj) // efficient combination of efficient parts
   {
    Swap(a,obj.a);
    Swap(b,obj.b);
    Swap(c,obj.c);
   }
 };

But what the most important, Resource Host Types are swapable. 

class H : NoCopy
 {
  private:

   ResourceHandle h; // Small Data Type

  public: 

   H() { SetNull(h); }

   H(A a,B b,C c) { h=New(a,b,c); }

   ~H() { Delete(h); }

   // swap objects

   void objSwap(S &obj) // efficient
    {
     Swap(h,obj.h);
    }
 };

That is the main cause why Swap is applicable to the wide range of types.

How to use

template <class T>
void CopySwap(T &a,T &b) noexcept(EnableNoExcept);

template <class T>
void Swap(T &a,T &b) noexcept(EnableNoExcept);

Well, you already known how to use Swap (see code above). Here is some implementaion details. The function Swap() uses the method objSwap() of the destined type to perform the operation. So to make a class swapable just define this method in the class. If a type is not a class type or the class has no this method, Swap() uses CopySwap() to do the job. In such case the type must be copyable. This is how Swap() is working on all basic types.

If for some reason you cannot or wantnot add the method to the class definition, you may specialize the struct SwapAdapters for your type: 

namespace N {

class C
 {
  ....
 };

}

namespace CCore {

template<>
struct SwapAdapters<false,N::C>
 {
  static void Swap(N::C &a,N::C &b) { .... }
 };

}

But there is a problem with this way: usually to implement Swap you need access to the private members of the class, so it might be impossible to implement Swap without an intervening in the class definition.
