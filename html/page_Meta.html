Meta

Files CCore/inc/gadget/Meta.h CCore/src/gadget/Meta.cpp

This file contains a collection of meta-tools. Meta-programming is a programming in compile-time. Sometimes you need to derive a constant, type or function based on the argument type properties. Meta supports solutions of such kind of tasks. It is enclosed in the CCore::Meta namespace.

Meta-programming produces its results as Meta-classes, i.e. a classes with public static members, like constants, inner types or static functions. Consider Meta-class as a compound return value, comprised by several values. If such class has some main return value, it is usually named as Ret or RetType.

Meta-function is a class template or template typedef, which, been instantinated, gives some Meta-class. Instantination of Meta-function is a "compile-time call" of it.

In some cases Meta-function is a template typdef, which produces a single resulting type.

Inheritance is used to include a result in a Meta-function.

Empty

Empty is an empty struct. It is used to return a no-value. Usually it is generated if a meta-function cannot produce a resonable result, i.e. in the case of the out of domain of definition call. 


struct Empty
 {
 };

DefType

Meta-function DefType returns its argument as a type: 


template <class T>
struct DefType
 {
  typedef T Ret;
 };

//
// typename DefType<T>::Ret == T
//

DefConst

Meta-function DefConst returns its second argument as a constant, first argument defines the constant type: 


template <class SUInt,SUInt Val> 
struct DefConst
 {
  enum RetType : SUInt { Ret = Val };
 };

//
// DefConst<T,Val>::Ret == Val
//

Booleans

True, False, If and IfNot produce a boolean result: 


/* struct True */

struct True 
 {
  enum RetType { Ret = true };
 };

/* struct False */

struct False 
 {
  enum RetType { Ret = false };
 };

/* struct If<bool Cond> */ 

template <> 
struct If<true> : True {}; 
 
template <> 
struct If<false> : False {};

/* struct IfNot<bool Cond> */ 

template <> 
struct IfNot<true> : False {}; 
 
template <> 
struct IfNot<false> : True {};

Select

Select is the one of the most important Meta-functions, it produces one of the given type based on the given condition: 


template <bool Cond,class T1,class T2>
using Select = typename ImpSelect<Cond,T1,T2>::Ret ;

//
// Select<true,T1,T2> == T1
//
// Select<false,T1,T2> == T2
//

Select is widely used in recursive meta-programming. It is implemented using template typedef, that eliminate the neccessity of keyword template.

BuildIf

BuildIf is another conditional tool: it performs instantination if the condition is true. 


template <bool Cond,template <class> class F,class T> struct BuildIf;

template <template <class> class F,class T>
struct BuildIf<true,F,T> : F<T> {};

template <template <class> class F,class T>
struct BuildIf<false,F,T> {};

//
// BuildIf<true,F,T> inherits F<T>
//
// BuildIf<false,F,T> is empty
//

Has

Has is used to determine a property of the type. 


template <class T,class ProbeSet>
struct Has
 {
  enum RetType { Ret = ProbeSet::template Probe<T>(0) };
 };

Here is an example. Assume you want to know if the given type has a non-static method named setInt of signature int(int). You can evaluate it using the following ProbeSet: 


struct ProbeSet_setInt
 {
  template <class T,int (T::*M)(int)> struct Host;
 
  template <class T>
  static constexpr bool Probe(...) { return false; }
  
  template <class T>
  static constexpr bool Probe(Host<T,&T::setInt> *) { return true; }
 };

Then apply Has to get the result: 


//
// Has<T,ProbeSet_setInt>::Ret
//

EnableIf

EnableIf allows poor simulation of concepts — the feature still missing in C++, it can be used to enable or disable a function instantination. 


template <bool Cond,class RetType=int>
using EnableIf = typename Select<Cond, DefType<RetType> , Empty >::Ret ;

Usually it is used to specify a function return value: 


template <class T>
EnableIf< SomeCond<T>::Ret , T  > SomeFunction(T arg);

It is also a template typedef, so you get the result without extra typing.

TypeListLen

TypeListLen is a length of the type list: 


//
// TypeListLen<>::Ret == 0
//
//
// TypeListLen<T1>::Ret == 1
//
//
// TypeListLen<T1,T2>::Ret == 2
//
// and so on
//

RangeObjType

RangeObjType is another template typedef, it gives the object type of range or poiner kind of types: 


struct SomeRange
 {
  T & operator * ();
 };

//
// RangeObjType<SomeRange> == T
//

IsSInt

IsSInt is true on signed integral types. By default, such types are signed char, short, int, long, long long and char if it is signed. But target may extend this list using specialization of the ExtraInt::Prop<T> structure in the PlatformBase.h header file. 


//
// IsSInt<T>::Ret == true if T is a signed integral type
//

IsUInt

IsUInt is true on unsigned integral types. By default, such types are unsigned char, unsigned short, unsigned, unsigned long, unsigned long long and char if it is unsigned. But target may extend this list using specialization of the ExtraInt::Prop<T> structure in the PlatformBase.h header file. 


//
// IsUInt<T>::Ret == true if T is an unsigned integral type
//

IsSUInt

IsSUInt is true if one of IsSInt or IsUInt is true, i.g. if the argument is an integral type. bool is not cosidered as integral. 


//
// IsSUInt<T>::Ret == true if T is an integral type
//

SIntToUInt

SIntToUInt returns an unsigned pair of the given signed integral type. It is empty if the type is not a signed integral. 


//
// SIntToUInt<T> is empty if IsSInt<T>::Ret == false
//
// typename SIntToUInt<T>::SType == "signed T"
//
// typename SIntToUInt<T>::UType == "unsigned T"
//

PromoteUInt

PromoteUInt returns a promote type for the given unsigned integral type. Promote type is a unsigend int, unsigend long or unsigend long long "greater or equal" than the original type. 


//
// PromoteUInt<T> is empty if IsUInt<T>::Ret == false
//
// typename PromoteUInt<T>::Type == "promote type of T"
//

PromoteSInt

PromoteSInt returns promote types for the given signed integral type. Signed promote type is a int, long or long long "greater or equal" than the original type. Unsigned promote type is a unsigend int, unsigend long or unsigend long long "greater or equal" than the original type. 


//
// PromoteSInt<T> is empty if IsSInt<T>::Ret == false
//
// typename PromoteSInt<T>::SType == "signed promote type of T"
//
// typename PromoteSInt<T>::UType == "unsigned promote type of T"
//

UIntBits

UIntBits returns the number of bits in the given unsigned integral type. 


//
// UIntBits<T>::Ret == "bit size of T"
//

HexWidth

HexWidth returns the number fo hex digits required for the type value representation. 


//
// HexWidth<T>::Ret == "hex size of T"
//

Target support for Meta

Target must provide the followig entities in its PlatformBase.h header file. 


namespace CCore {

namespace Bits {

const unsigned unsigned_char      = "bit size of type char" ;
const unsigned unsigned_short     = "bit size of type short" ;
const unsigned unsigned_int       = "bit size of type int" ;
const unsigned unsigned_long      = "bit size of type long" ;
const unsigned unsigned_long_long = "bit size of type long long" ;

} // namespace Bits

} // namespace CCore

The following struct is used to extend the list of types, recognized as integral types: 


namespace CCore {

namespace ExtraInt {

template <class T>
struct Prop 
 {
  enum 
   {
    IsSigned   = false,
    IsUnsigned = false
   };
 };

template <>
struct Prop<"extra sint type">
 {
  enum 
   {
    IsSigned   = true,
    IsUnsigned = false,
   };
  
  typedef "signed type" SType;
  typedef "unsigned type" UType;
   
  typedef "signed promote type" PromoteSType;
  typedef "unsigned promote type" PromoteUType;
 };

template <> 
struct Prop<"extra uint type">
 {
  enum 
   {
    IsSigned   = false,
    IsUnsigned = true,
      
    Bits = "bit size"
   };
  
  typedef "promote type" PromoteUType;
 };

} // namespace ExtraInt

} // namespace CCore
