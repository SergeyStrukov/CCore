XCore task events

XCore generates number of events during task execution from synchronization operations and task switches.

Task switch events

Task switch events are generated when task switch occurs or interupt occurs. Task is assigned an enum value of the TaskNumber type. The name, given to the task, is also assigned to the correspondent task number. Two special value: EnterInt and LeaveInt, are used to designate entry of the interrupt context and exit of the interrupt context. Task switches generates events of the TaskSwitchEvent type. 


struct TaskSwitchEvent
 {
  EventTimeType time;
  EventIdType id;
  
  uint16 task;
 };

task is the number of the task, which become active.

Mutex events

Mutex is assigned an enum value of the MutexNumber type. The name, given to the mutex, is also assigned to the correspondent mutex number. Mutex generates events of the MutexEvent type. 


struct MutexEvent
 {
  EventTimeType time;
  EventIdType id;
  
  uint16 task;
  uint16 mutex;
  uint8 type;
  
  enum Type : uint8
   {
    Lock,
    Unlock,
    IncLock,
    DecLock,
    Block
   };
 };

task is a task number.

mutex is a mutex number.

type is an event type. There are five event types.

Lock is generated when the mutex is locked by the task. The correspondent maker is Up.

Unlock is generated when the mutex is released by the task. The correspondent maker is Down.

IncLock is generated when the task increments the lock count of the mutex. The correspondent maker is Inc.

DecLock is generated when the task decrements the lock count of the mutex. The correspondent maker is Dec.

Block is generated when the task is blocked on the mutex. The correspondent maker is UpBlock.
